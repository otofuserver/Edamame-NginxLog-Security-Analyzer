# block_ip.html
対象: `src/main/resources/fragments/block_ip.html`

## 概要
- block_ipテーブルを管理する断片ページ。ステータス・種別でフィルターし、20件単位で一覧表示し、手動作成と削除（ミニメニュー＋確認モーダル）を提供する。閲覧は認証ユーザー全員、作成/削除はオペレーター以上。

## 主な機能
- ステータスフィルター: 有効/期限切れ/無効/すべて（既定: 有効、表示順は「すべて表示」を先頭）。
- 種別フィルター: すべて/モニターブロック/アプリログイン/マニュアル。
- 検索入力（IP・理由・作成者など）と右寄せの「手動作成」ボタン。
- テーブル表示（IP・種別・対象サーバー・ステータス・開始/終了・理由・更新）。20件ページング。
- ミニメニューで「有効化」「無効化」「削除」操作を提供し、どれも確認モーダルを経由（canOperate=trueのみ）。
- 手動作成モーダルは種別固定（マニュアル）で作成。

## 挙動の補足
- 初期ソートはIP昇順。ソート/フィルター/検索状態はURLクエリに保存され、リロード(F5)後も引き継がれる。

## 挙動
- 挿入後に `BlockIp.init()` を呼び出し、list_view_coreでページングしつつ `/api/block-ip` からデータ取得（閲覧は全認証ユーザー可）。
- ステータス未指定時もサーバー側は ACTIVE を既定として扱い、UIの初期値と合わせる。
- 「手動作成」で入力モ���ダルを開き、POST `/api/block-ip` を送信（serviceType=MANUAL, end_atは空欄または現在以降のみ、オペレーターのみ実行可能）。
- 行クリックでミニメニューを開き、「有効化」「無効化」「削除」を確認モーダル経由で実行（オペレーターのみ）。

## 細かい指定された仕様
- スクリプト依存: `/static/list_view_core.js`, `/static/mini_menu.js`, `/static/block_ip.js`。
- ページサイズは20固定（list_view_coreのdefaultSize）。
- ミニメニューは「有効化」「無効化」「削除」を提供し、確認モーダルあり。canOperate=false時は無効。
- フィルター状態はSTATEに保持しつつURLクエリへ反映し、リロードしても復元する。

## その他
- HTMLのみでロジックは含まず、JS初期化に依存。スタイルは既存のtable/card共通クラスを使用。
