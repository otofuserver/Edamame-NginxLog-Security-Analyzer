# logout / ログアウト UI

対象: クライアント側ログアウトボタンおよびモジュール（`src/main/resources/static/logout.js` に対応する説明）

## 概要
- サイドバー等に存在するログアウト操作に関するクライアント側振る舞いとセキュリティ要件を記載する。

## 主な機能
- ログアウトボタンの押下ハンドラ登録
- サーバへログアウトリクエスト（POST または GET）送信
- CSRF トークン処理（必要な場合）

## 挙動
- ボタン押下で `/logout` エンドポイントへリクエストを送信し、成功時は `/login?logout=success` にリダイレクトする。
- セッション破棄後は保護されたページへアクセスすると `/login` にリダイレクトされる。

## 細かい指定された仕様
- CSRF 対策が有効な場合、ログアウトリクエストには有効な CSRF トークンを付与すること。
- クライアントは `window.Logout.init()` のような初期化関数でイベントバインドを行う。
- ログアウト処理は idempotent（多重押下による副作用が発生しない）であることが望ましい。

## その他
- テスト手順: ログイン状態で logout ボタンを押し、`/login?logout=success` へ遷移すること、保護ページへアクセスすると `/login` にリダイレクトされることを確認。
